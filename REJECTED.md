# Rejected paths

Read this before proposing alternatives. These ideas were considered and rejected during design — the reasoning is here so they aren't re-litigated.

## Conventional layout patterns

Six conventional layouts were sketched first: kanban-by-area, personal-first stack, two-column lobby/work, sidebar+main, inbox+detail, single chronological feed. They were rejected as too conventional. All six made the same hidden assumption — that the dashboard is a *container of work artifacts* and the layout question is "how do we arrange the artifacts." The reframe to "office space with rooms" came after these were rejected.

The point isn't that they're bad layouts. The point is the dashboard is not a list of cards. It's a digital office/workspace.

## Vision as a defined UI element

Several patterns were proposed for surfacing the project vision in the lobby — quote wall, "why I'm here" strip per teammember, scrapbook of references, rolling weekly question. All were rejected as "show don't tell."

Vision is undefined-but-important. It needs to *emerge* from the work and the place, not be authored or pinned. There is no vision UI element. This is not tabled forever — it's tabled until something inspires the right form.
